구성도에 반드시 표기할 항목 (입력 전 점검)

  1. 망 영역 경계 — 내부업무망 · 인터넷(외부)망 · DMZ · 전산실(서버팜) · DB존 · 개발망을 구획선으로 구분
  2. 인터넷 접점 — 외부망과 만나는 모든 지점과 경로(회선 포함)
  3. 보안장비 위치 — 방화벽(외부·내부), IPS/IDS, VPN, NAC 등의 설치 지점
  4. 망분리 방식 표기 — 서버·단말별 물리적/논리적(CBC·SBC) 분리 여부 명시
  5. 망간 자료전송 구간 — 망연계 솔루션 위치와 전송 방향
  6. 단말 구성 — 업무용 PC와 인터넷용 PC(또는 가상화 영역)의 소속 망
  7. 대외 공개 서비스 — 홈페이지·인터넷뱅킹 등 공개서버의 위치(DMZ)
  8. 원격접속 경로 — 재택·외부 접속 시 VPN 게이트웨이와 인증 방식
  9. 무선망 — AP 존재 여부, 연결되는 망, 인증·암호화 방식
  10. 클라우드 · SaaS 연결 — 이용 중인 외부 서비스와 접속 경로·통제 지점
  11. 대외기관 연계 — 금융결제원·VAN 등 전용회선 연계 구간
  12. 범례와 기준일 — 기호 설명, 작성일, IP 대역(세그먼트) 표기

권고 표준 구성도

입력하신 구성을 기준으로, 규정을 충족하도록 보완한 목표 구성도입니다. 주황 점선은 추가, 적색 점선은 제거·차단이 필요한 요소입니다.

기존 구성 유지 추가 필요 제거·차단 필요
본 도구는 구성도 자체점검을 돕는 참고용이며, 법적 판단을 대신하지 않습니다. 최종 준수 여부는 전자금융감독규정·동 시행세칙 원문, 금융보안원 「금융부문 망분리 가이드라인」 및 보안해설서 최신본, 감독당국 유권해석에 따라 확인하시기 바랍니다. 조항 번호는 개정으로 변경될 수 있으므로 인용 전 원문 대조가 필요합니다. 입력한 파일과 응답 내용은 이 브라우저 안에서만 처리됩니다(AI 자동 인식을 직접 실행한 경우 해당 게이트웨이로만 전송).
내부망(폐쇄망) 배포 안내
① 이 HTML 파일 1개를 내부 웹서버(예: nginx, IIS) 문서 폴더 또는 사내 공유 위치에 복사합니다. 별도 설치·DB·외부 통신이 없습니다.
② PDF 미리보기를 쓰려면 pdf.js 배포본에서 pdf.min.js, pdf.worker.min.js 두 파일을 받아 이 파일과 같은 위치의 lib/ 폴더에 두십시오. (없어도 검토 기능은 전부 동작하며, PDF는 이미지로 변환 후 업로드를 권장)
③ AI 자동 인식은 선택 기능입니다. 사내에 Anthropic Messages API 호환 게이트웨이가 있으면 제2절의 설정에 주소를 입력하십시오. 인터넷이 차단된 환경에서 외부 주소를 입력하면 당연히 실패하며, 수동 식별로 동일한 검토가 가능합니다.
④ 규정 개정 시 파일 상단 RULES(자바스크립트) 배열의 문구만 수정하면 됩니다.
